Flexbox

  • Resumo: - Definir um recipiente flex; inline ou block, dependendo do valor dado. Permite um contexto flex para todos os seus filhos diretos, com diferentes tratamentos para aquele flex item.

  1. flex-direction: - Isso estabelece o eixo principal, definindo assim a direção que os flex items são colocados no recipiente flex. Flexbox é um conceito de layout de direção única. Pense nos flex items como principalmente colocados em linhas horizontais ou colunas verticais.

    Exemplo: - flex-direction: row | row-reverse | column | column-reverse;

    1. row (padrão); da esquerda (left) para a direita (right) ltr; direita (right) à esquerda (left) rtl;

    2. row-reverse; direita (right) para a esquerda (left); da esquerda (left) para a direita (right).

    3. column; o mesmo que row, mas de cima (top) para baixo (bottom).

    4. column-reverse; o mesmo que row-reverse, mas de baixo (bottom) para cima (top).

  2. flex-wrap: Por padrão, os flex items tentarão todos se encaixar em uma linha. Você pode mudar isso e permitir que os itens embrulhem conforme necessário com esta propriedade.

    Exemplo - flex-wrap: nowrap | wrap | wrap-reverse;

    1. nowrap (padrão): Todos os flex items estarão em uma linha

    2. wrap; Os flex items serão embrulhado em várias linhas, de cima a baixo, e para quebra de linha caso ele não couber.

    3. wrap-reverse; Os flex items envolverão em várias linhas de baixo para cima.

  3. flex-flow: Esta é uma abreviação de propriedades de flex-direction e flex-wrap, que juntas definem os eixos principal e cruzado do flex container. O valor padrão é row nowrap.

    Exemplo - flex-flow: column wrap;

  4. justify-content: Isso define o alinhamento ao longo do eixo principal. Ele ajuda a distribuir sobras extras de espaço livre quando todos os flex items em uma linha são inflexíveis, ou são flexíveis, mas atingiram seu tamanho máximo. Ele também exerce algum controle sobre o alinhamento dos itens quando eles transbordam a linha.

    justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly | + safe | unsafe;

    1. flex-start (padrão); Os itens são colocados no início do flex-direction.

    2. flex-end; Os itens são colocados no final do flex-direction.

    3. start; Os itens são colocados para o início da direção writing-mode.

    4. end; Os itens são colocados no final da direção writing-mode.

    5. center; Os itens são centrados ao longo da linha.

    6. space-between; Os itens são distribuídos uniformemente na linha; primeiro item está na linha de partida (start), último item na linha final (end).

    7. space-around; Os itens são distribuídos uniformemente na linha com espaço igual ao seu redor. Note que visualmente os espaços não são iguais, já que todos os itens têm espaço igual em ambos os lados. O primeiro item terá uma unidade de espaço contra a borda do contêiner, mas duas unidades de espaço entre o próximo item porque o próximo item tem seu próprio espaçamento que se aplica.

    8. space-evenly; Os items são distribuídos para que o espaçamento entre os dois items (e o espaço até as bordas) seja igual.

  5. align-items: Isso define o comportamento padrão de como os itens flex são dispostos ao longo do cross axis na linha atual. Pense nisso como justify-content da versão para o eixo transversal (perpendicular ao eixo principal (main-axis)).

    align-items: stretch | flex-start | flex-end | center | baseline | first baseline | last baseline | start | end | self-start | self-end + ... safe | unsafe;

    1. stretch (padrão); Estica os elementos para encher o recipiente (ainda respeitando a largura mínima (min-width) / largura máxima (max-width) ).

    2. flex-start / start / self-start; Os itens são colocados no início do eixo cruzado (cross axis). A diferença entre estes é sutil, e é sobre respeitar as regras flex-direction ou as regras writing-mode.

    3. flex-end / end / self-end; Os itens são colocados nas extremidades do eixo cruzado (cross axis). A diferença novamente é sutil e é sobre respeitar regras flex-direction versus writing-mode.

    4. center; Os items estão centrados no eixo transversal

    5. baseline; Os items estão alinhados, como base em suas linhas de base alinhadas.

  6. align-content: Alinha as linhas de um recipiente dentro do flex quando há espaço extra no eixo transversal (cross axis), semelhante à forma como (justify-content) alinha items individuais dentro do eixo principal (main-axis).

    align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch | start | end | baseline | first baseline | last baseline + ... safe | unsafe;

    1. normal (padrão); Os items são colocados em sua posição padrão como se nenhum valor fosse definido.

    2. flex-start / start; Os items são colocados até o início do recipiente. O (mais suportado) flex-start segui o flex-direction enquanto start segui a direção writing-mode.

    3. flex-end / end; Os items são colocados até o final do recipiente. O (mais suportado) flex-end segui o flex-direction enquanto o final (end) segui a direção writing-mode.

    4. center; Os items centralizados no recipiente

    5. space-between; Os items são distribuídos uniformemente; a primeira linha é no início do recipiente, enquanto o último está no final.

    6. space-around; Os items são distribuídos uniformemente com espaço igual em cada linha.

    7. space-evenly; Os items são distribuídos uniformemente com espaço igual ao seu redor.

    8. stretch;Os items são alongados para ocupar o espaço restante.

    Nota: Esta propriedade só tem efeito em recipientes flexíveis de várias linhas, onde (flex-flow) está definido para (wrap) ou (wrap-reverse)). Um recipiente flexível de linha única (ou seja, onde (flex-flow) está definido para o seu valor padrão (no-wrap)), não refletirá no (align-content).

  7. order: Por padrão, os flex items estão colocados na ordem de origem. No entanto, a propriedade order controla a ordem em que eles aparecem no recipiente (flex container).

    order: 5; -- O padrão é 0 --

  8. flex-grow: Isso define a capacidade de um flex item crescer, se necessário. Aceita um valor sem unidade que serve como proporção. Ele dita a quantidade do espaço disponível dentro do recipiente (container) que o flex item deve ocupar.

    Se todos os items tiverem sido definidos como 1, o espaço restante no container será distribuído igualmente para todas as crianças. Se uma das crianças tem um valor de 2, o espaço restante ocuparia o dobro de espaço que os outros (ou tentará, pelo menos).

    flex-grow: 4; /* o padrão é 0 *

    Nota: Os números negativos são inválidos.

  9. flex-shrink: Isso define a capacidade de um flex item se encolher, se necessário.

    flex-shrink: 3; /* Por padrão é 1 */

    Nota: Os números negativos são inválidos.

  10. flex-basis: Isso define o tamanho padrão de um elemento antes que o espaço restante seja distribuído. Pode ser um comprimento (por exemplo, 20%, 5rem, etc.) ou uma palavra-chave. A palavra-chave auto significa "olhe para a minha propriedade de largura ou altura" (que foi temporariamente feita pela palavra-chave main-size até ser rejeitada). A palavra-chave content significa "dimensioná-lo com base no conteúdo do item" – esta palavra-chave ainda não é bem suportada, por isso é difícil testar e mais difícil saber o que seus irmãos max-content, min-content e fit-content fazem.

    Se definido para 0, o espaço extra em torno do conteúdo não é contabilizado. Se definido para auto, o espaço extra é distribuído com base em seu valor flex-grow.

  11. flex: Esta é a taquigrafia para flex-grow, flex-shrink e flex-basis combinados. O segundo e o terceiro parâmetros ( flex-shrink e flex-basis ) são opcionais. O padrão é 0 1.

    flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]

    Recomenda-se que você use esta propriedade taquigrafia em vez de definir as propriedades individuais. A taquigrafia define os outros valores de forma inteligente.

  12. align-self: Isso permite que o alinhamento padrão (ou o especificado por align-items ) seja substituído por flex items individuais.

    align-self: auto | flex-start | flex-end | center | baseline | stretch;

    Nota que (float), (clear) e (vertical-align) não tem efeito sobre um flex item.
